## How to Update This File

When making a new release, follow these guidelines:

1. **Version Format**: Use semantic versioning (MAJOR.MINOR.PATCH)
2. **Section Format**: Use the categories above (Added, Changed, Deprecated, Removed, Fixed, Security)
3. **Chronological Order**: List changes in reverse chronological order (newest first)
4. **Keep Unreleased Section**: Always maintain an [Unreleased] section at the top
5. **Release Date Format**: Use YYYY-MM-DD format for dates
6. **Link Format**: Update the bottom links section with the new version

Example entry:
```markdown
## [X.Y.Z] - YYYY-MM-DD

### Added
- New feature description

### Fixed
- Bug fix description
```
